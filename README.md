# 🛠 기술 스택 및 도구 
- HTML & CSS: 웹 페이지의 기본 구조와 스타일을 정의합니다. HTML은 웹사이트의 골격을 제공하며, CSS는 이를 스타일리시하고 반응형으로 만드는 데 사용됩니다.
- JavaScript: 사용자와의 상호작용을 강화하고 동적 요소를 웹 페이지에 통합하여 사용자 경험을 향상시킵니다.
- PHP: 서버 측 로직을 처리하고 데이터베이스와의 연동을 담당합니다. 비동기 방식을 활용하여 사용자 요청에 대해 신속하고 효율적으로 응답합니다.

### 경쟁사 장점
- 마라톤 대회 일정 전체가 날짜순으로 정리되어 있음
- 검색도구를 통해 내가 원하는 마라톤을 따로 알아볼 수 있음
- 마라톤 교실 페이지를 통해 관련된 다양한 기초 지식 정보를 알려줌

### 경쟁사 단점
- 오래된 웹사이트인 만큼 디자인이 올드함
- 웹사이트를 들어갔을 때 직관적이기 보단 조잡한 느낌이 강함
- 페이지의 대부분이 글로 이루어져 있어 가독성이 부족함
- 관리되지 않는 불필요한 카테고리가 존재함

# 🏃‍♀️ 마라톤 시장 규모 조사
![image](https://github.com/HwangInJi/Way-Run-Meaning/assets/163365140/c2c42ccc-e6e0-47e1-9d60-d586c13e57dd)

- 2021서울마라톤조직위원회에 따르면 2016년 8382명(31.1%)이던 20, 30대 참가자가 5년 후인 2021년에는 1만966명(64.4%)까지 늘어났을 만큼
20-30세대의 러닝인구가 증가하고 있는 추세
- 코로나의 영향으로 러닝화만 있다면 언제 어디서든 시작할 수 있다는 점이 낮은 진입장벽을 만들어 접근성을 높여주는 데 기여함 
- 이들 사이에 자리잡힌 SNS를 통해 정보를 공유하는 문화가 더 나아가 개인의 휴대전화을 활용해 자신이 원하는 장소에서 달리기를 한 후 기록을 인증하는 방식이 유행처럼 번지게 되면서
자연스럽게 러닝이 바이럴화 됌

# 💡 프로젝트 네이밍 정의
### Way Run Meaning(웨이런미닝)
1. 'Way : 방법 / Run : 달리다 / Meaning : 의미' 라는 세 단어를 조합하여 <strong>의미있게 달리는 방법을 알려주는 곳</strong>이라는 뜻을 담고 있음
2. '무한도전' 정형돈의 유행어 중 하나인 웨이런미닛~ 이라는 밈에서 착안, 주 타겟이 2030인 만큼 연결되는 발음을 이용하여 재미요소를 담고자 함
<br><br>

# ✨ 프로젝트 로고 및 캐릭터 정의
![image](https://github.com/HwangInJi/Way-Run-Meaning/assets/163365140/26ae4271-d2e8-4084-aac1-b4462e6b29aa)
1. 로고
- 러닝이 주는 성취감을 담은 디자인으로, 메탈질감의 뱃지 형태로 제작되어 뿌듯함과 성취감을 함축적으로 전달
- 꽃의 형상을 활용하여 러닝을 통해 긍정적인 모습을 피워나가는 과정을 상징적으로 나타냄
<br>

2. 캐릭터   
- 다양한 마라톤 대회 정보 및 커뮤니티 서비스를 제공하면서 브랜드의 일관된 아이덴티티를 유지하기 위해, 각 서비스 카테고리를 하나의 디자인 컨셉으로 통일시키기 위한 캐릭터를 개발   
- 캐릭터는 눈을 감고 있는 모습으로, 러닝의 즐거움을 편안함으로 해석하여 안정감과 편안함을 시각화
- 화이팅 넘치는 의욕적인 손동작을 통해 같은 목표로 달려가는 사람들을 상징적으로 표현
<br><br>

# 🖼️ 메인 디자인 컨셉
![image](https://github.com/HwangInJi/Way-Run-Meaning/assets/163365140/586ce4ee-bead-4884-9e85-8ccd3820aff1)
1. 디자인 컨셉
- 현대적이고 트렌디한 러닝 커뮤니티를 컨셉으로 MZ세대에게 매력적으로 다가가며, 러닝을 더욱 힙하고 즐거운 경험으로 전환
- 이를 통한 오래동안 머물고 싶은 웹 러닝 커뮤니티 형성
- 글로벌하고 역동적인 분위기 -> 현대적이고 감각적인 일러스트 그래픽 요소 캐릭터를 통해 사용자들이 참여하고 소통할 수 있는 다양한 기능들을 제공
<br>

![image](https://github.com/HwangInJi/Way-Run-Meaning/assets/163365140/c7c77895-9376-4f29-8e83-b0ac3cd7390f)
3.컬러
- 오렌지 컬러는 활기찬 느낌을 전달하면서도 긍정적인 에너지를 부여 -> 전반적인 건강과 정서적 에너지를 증진시키며 회복하는 데 도움이 되는 컬러 
- 이는 러닝의 즐거움과 창의성을 강조하는 웹사이트의 콘텐츠와 잘 부합하며 오렌지 컬러는 따뜻함과 열정을 상징하기 때문에 러너들 사이에서 친근하고 활기찬 분위기를 조성
- 따라서 오렌지를 주요 컬러로 선택함으로써 MZ세대에게 힙하고 현대적인 이미지를 전달
- 오렌지 컬러는 기쁨과 창의성의 색으로 알려져 이있으며 이는 따뜻함, 열기, 햇빛, 열정, 창의성, 성공, 격려, 변화, 결단, 건강, 자극, 행복, 재미, 즐거움, 균형, 자유상징
<br>
- 볼드한 산세리프체로 러닝 커뮤니티의 역동적이고 강렬한 분위기를 강조하기 위함
- 산세리프체는 단호하고 힘있는 느낌을 전달하면서도 현대적이고 감각적인 이미지를 구축할 수 있음 -> 읽기 쉽고 명확하게 사용자들에게 정보를 전달하는 데 효과적

# 🛠️ 사용한 기술/기능 설명

## 1. 회원가입 
- 회원가입 폼에서의 사용자 편의성을 높이기 위해 중복확인 버튼을 제거하고, 다음과 같이 비동기 방식을 선택했습니다. 이를 통해 사용자가 더욱 원활하고 직관적으로 접근할 수 있도록 구현하였습니다.

![image 48](https://github.com/nicejmp1/Way-Run-Meaning/assets/163364733/6f346139-fb09-43bc-9e93-fc7c517941f2)

- 회원가입 과정에서 사용자의 편의를 극대화하기 위해 다음과 같이 비동기 처리 방식을 적용하였습니다. 이메일과 닉네임 입력 시, 시스템은 실시간으로 해당 정보의 중복 여부를 확인하고, 사용 중인 경우 즉시 사용자에게 그 사실을 알립니다. 이는 사용자가 폼을 제출한 후에야 결과를 알 수 있는 기존 방식보다 훨씬 직관적이고 효율적입니다. 비밀번호 입력에 대해서는 유효성 검사를 통해 안전한 비밀번호 사용을 유도하고, 사용자가 입력하는 즉시 비밀번호가 일치하는지 여부도 확인할 수 있도록 하여, 사용자 경험을 향상시키는 데 중점을 두었습니다.

<br> 

## 2. 아이디 찾기와 비밀번호 찾기 
- 아이디 찾기와 비밀번호 찾기 기능을 사용자 친화적으로 구현하였습니다. 아이디 찾기에서는 사용자가 제공한 이메일과 이름이 데이터베이스의 기록과 일치할 경우 계정이 존재한다는 안내 메시지를 표시하고, 일치하지 않을 경우 회원가입을 유도합니다. 이는 사용자에게 명확한 지침을 제공하여 효율적인 서비스 이용을 가능하게 합니다. 

![Frame 65](https://github.com/nicejmp1/Way-Run-Meaning/assets/163364733/bc5e4296-df75-4723-9d5b-4e10007470ab)

- 비밀번호 찾기에서는 사용자의 이메일과 이름이 일치하는지 확인 후, 일치한다면 보안을 고려하여 인증 메일을 발송합니다. 이메일에 포함된 토큰은 5분의 유효 시간을 갖고, 사용자가 제시간 내에 인증 코드를 입력하면 비밀번호를 재설정할 수 있는 권한을 부여받습니다. 이 절차는 사용자의 계정 보안을 강화하면서 동시에 비밀번호 재설정을 원활하게 진행할 수 있도록 설계되었습니다.

<br>

## 3. 대회 일정 페이지
- 대회 일정 페이지는 사용자가 편리하게 정보를 접할 수 있도록 카테고리 이동과 페이지 전환 시 UX를 개선하여 화면의 깜빡임을 최소화하였습니다. 이러한 설계는 사용자의 원활한 탐색을 돕고 정보 접근성을 향상시킵니다.

![Frame 66](https://github.com/nicejmp1/Way-Run-Meaning/assets/163364733/6d522960-4350-4a7b-aac3-f193c2b31cd1)

추가로, 사용자가 관심 있는 일정을 쉽게 저장하고 관리할 수 있도록 '좋아요(찜하기)' 기능을 도입하였습니다. 이 기능을 사용하면 선택한 일정이 마이페이지로 전송되어 나중에도 쉽게 찾아볼 수 있습니다. 또한 각 일정의 좋아요 수를 표시하여, 해당 일정이 다른 사용자들 사이에서 얼마나 인기가 있는지를 직관적으로 확인할 수 있도록 하였습니다. 이러한 요소들은 사용자에게 유용한 정보를 제공하고, 사이트의 상호작용성을 높이는 데 기여합니다.

<br>

## 4. 마이페이지
- 마이페이지는 사용자의 개인 정보와 관련 활동을 직관적으로 보여주기 위해 세심하게 설계되었습니다. 사용자가 회원가입한 날짜를 기준으로 D-day 카운트를 제공하여, 가입 후 지난 시간을 한눈에 확인할 수 있습니다. 또한 사용자가 소속된 크루가 있다면, 해당 크루의 이름을 마이페이지에 표시하여 소속감을 높이고 커뮤니티 활동을 촉진합니다.

![Frame 67](https://github.com/nicejmp1/Way-Run-Meaning/assets/163364733/f0f210bf-6908-448d-aed3-d9eae0415bf7)

사용자가 자신을 표현할 수 있도록 마이페이지에는 사용자가 선택한 이미지를 메인으로 표시하며, 이미지 아래에는 사용자의 닉네임과 이메일 주소를 명확하게 보여주어 개인 정보에 쉽게 접근할 수 있도록 하였습니다. 이러한 디자인은 사용자의 편의성을 우선하면서도 개인화된 경험을 제공하는 데 중점을 두었습니다.

<br>

## 5. 마이페이지에 캘린더 
- 마이페이지에 캘린더 기능을 통합하여 사용자가 찜한 대회 일정을 날짜별로 직관적으로 확인할 수 있도록 구현하였습니다. 이 캘린더는 사용자가 관심을 표현한 일정을 시각적으로 쉽게 파악할 수 있게 도와주며, 사용자의 계획을 효율적으로 관리할 수 있도록 지원합니다.

![image 62](https://github.com/nicejmp1/Way-Run-Meaning/assets/163364733/6a45d4fd-00ed-4c16-981a-971a02b45c75)


캘린더 옆에는 사용자가 찜한 일정의 리스트를 별도로 보여주어, 각 일정에 대한 빠른 접근성을 제공합니다. 사용자가 이 리스트에서 특정 일정을 선택하면, 해당 일정과 관련된 상세 페이지나 외부 사이트로 직접 이동할 수 있는 링크를 제공하여, 추가 정보를 편리하게 탐색할 수 있도록 하였습니다. 이러한 기능 구성은 사용자 경험을 풍부하게 하며, 웹사이트의 사용성을 높이는 중요한 요소로 작용합니다.

## 7. 커뮤니티(게시판) 페이지
- 웹사이트 내에서 커뮤니티 활성화를 위해 네 가지 주요 게시판을 구현하였습니다: 자유게시판, 크루모집, 번개모임, 질문있어요. 각 게시판은 사용자가 자유롭게 글을 작성, 저장, 수정할 수 있는 기능을 지원합니다. 이는 사용자 간의 상호작용을 촉진하고, 다양한 주제에 대해 정보를 공유하고 토론할 수 있는 공간을 제공합니다.

![6](https://github.com/HwangInJi/Way-Run-Meaning/assets/163365140/53cc19b1-3f46-4285-9152-b8b1755fdf3a)

- 관리자가 작성한 글은 커뮤니티 내에서 중요한 정보로 인식될 수 있도록 최상단에 고정하고, '공지'라는 태그를 명확하게 표시하여 사용자가 쉽게 인식할 수 있도록 하였습니다. 이러한 접근 방식은 중요한 정보의 노출을 극대화하고, 사용자의 주의를 끌어 정보 전달의 효율을 높입니다.

![7](https://github.com/HwangInJi/Way-Run-Meaning/assets/163365140/d5e822ae-a080-4044-870f-bb3a8a5c859b)

- 또한, 각 게시글에 댓글 기능을 추가하여 사용자 간의 의견 교환을 용이하게 하였습니다. 사용자는 댓글을 자유롭게 작성, 수정, 삭제할 수 있어, 활발한 토론과 상호작용이 이루어질 수 있습니다. 이 모든 기능은 사용자 경험을 향상시키며, 커뮤니티의 활동적인 참여를 장려합니다.
